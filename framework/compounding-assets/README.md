# Compounding assets

What actually compounds over long periods of time.

The core product question: **"What is actually compounding?"**

---

## Asset categories

| Asset | Compounds when… | Decays when… |
|-------|-----------------|--------------|
| **Trust** | Promises kept, consistency, reliability over years | Overpromising, reputation damage, broken commitments |
| **Systems** | Reusable infrastructure, documented processes, automation | One-off hacks, tribal knowledge, no documentation |
| **Institutional memory** | Decisions, rationale, and patterns preserved | Turnover, no decision journal, repeated mistakes |
| **Capital** | Reinvested, allocated toward ownership | Consumed, prestige-spent, no asset accumulation |
| **Distribution** | Audience, channels, network effects built | Renting platforms, no owned channels, algorithm dependency |
| **Relationships** | Deep, mutual, long-term | Transactional networking, shallow connections |
| **Health** | Sustainable energy, recovery systems | Burnout cycles, unsustainable pace |
| **Judgment** | Decision journal, outcome review, pattern recognition | Same mistakes repeated, no reflection loop |
| **Reputation** | Consistent quality, earned credibility | Short-term optimization, prestige chasing |
| **Reusable infrastructure** | Code, content, templates, frameworks | Starting from zero each project |

---

## Observable signals

**Compounding present:**
- Reusable systems emerge from work
- Trust deepens with key relationships
- Assets accumulate (code, audience, capital, knowledge)
- Decisions reference prior rationale
- Projects build on prior projects

**Compounding absent:**
- Every project starts from zero
- High activity, low asset accumulation
- Knowledge lives only in memory
- Networking without ownership
- Learning without application or assets
