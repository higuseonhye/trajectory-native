# Compounding Analysis

Surface what is actually compounding — reusable systems, trust, capital, distribution, relationships, judgment, assets.

---

## Core question

> "What is actually compounding?"

---

## Signals to measure

| Signal | Indicates compounding when… |
|--------|----------------------------|
| Reusable systems | Work products used across projects |
| Trust depth | Relationship quality increasing over time |
| Asset accumulation | Code, audience, capital, infrastructure growing |
| Distribution ownership | Owned channels vs rented platforms |
| Institutional memory | Decisions referenced, patterns recognized |
| Project reuse | New work builds on prior work |
| Leverage ratio | Output growing faster than time input |

---

## Anti-signals (non-compounding)

- Every project starts from zero
- High networking, low ownership outcomes
- Learning without application
- Tool accumulation without systems
- Activity metrics rising, assets flat

---

## Product behavior

1. Analyze trajectory events over time windows
2. Classify activity as compounding vs consumptive
3. Surface leverage accumulation trends
4. Contrast stated goals with behavioral signals
5. Recommend trajectory adjustments — not motivational advice
