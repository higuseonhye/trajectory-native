# Context-embedded calibration

**Source:** trajectory-notes (2026-05-15, 2026-05-16)  
**Type:** product-mapping — interaction design

---

## Core insight

Product is not anti-conversation — it is anti **conversation outside context**.

High-signal interaction already happens on X, in DMs, at accelerators, on GitHub issues. Calibration often **disappears** when the conversation ends.

---

## Design principle

> Back-and-forth should happen **inside context** — where adjustment, calibration, and shared progress occur.

Calibration threads attach to obsession fields, feed pivots, and reasoning traces — not a global inbox.

---

## Coordination fragmentation

AI builders ship faster. Context still lives in Slack threads, Notion pages, and README sections that drift out of sync.

**Question:** Can a trajectory-native workspace reduce re-explaining context to collaborators and future-self?

**Signal to watch:** Reasoning traces linked back to when decisions were made — not just written once.

---

## Product surfaces

| Surface | Role |
|---------|------|
| Calibration threads | Context-anchored replies |
| Commit context blocks | Execution ↔ reasoning linkage |
| Decision journal | Personal judgment compounding |
| Decision memory (org-reasoning-mvp) | Team institutional memory |
